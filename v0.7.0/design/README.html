Design records

Durable rationale for load-bearing decisions. A DR records why, so future contributors change decisions knowingly rather than accidentally. DRs are immutable once accepted; a reversal is a new DR that supersedes the old one.

Format: see TEMPLATE.md. IDs are DR-NNNN, allocated sequentially, never reused.
